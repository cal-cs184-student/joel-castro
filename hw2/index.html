Overview

In the following writeup I describe my jorney to creating a working MeshEdit tool. At a high-level, this means creatings modems for displaying 3D geometry. For instance, using bezier control-points to define 3D curves or "patches", or a traingle-mesh-based editor that uses edge splitting and manipulation to implement a subdivision scheme that gets meshes to appear at desired smoothness. I am really happy with the insights I gained from the project on data structures used to represent geometry (efficiently). Get ready to deal with a lot of pointers!

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

Castelijau is an algorithm for linearly interpolating every point on the Bezier curve corresponding to a set of control points. My implementation was constructed using the following logic: Given a set of control points: \begin{equation} \mathcal{C} = \{ c_1, c_2, \dots, c_n \}, \quad c_i \in \mathbb{R}^d \end{equation} we assume they are ordered such that: \begin{equation} x_1 \leq x_2 \leq \dots \leq x_n, \quad \text{where } c_i = (x_i, y_i). \end{equation} We compute the set of Bézier curve points, \( \mathcal{B} \), using the following procedure:
  • \textbf{Initialization:} \( \mathcal{B} \gets \emptyset \).
    • \textbf{For each parameter \( t \in (0,1) \), perform De Casteljau's algorithm:} \begin{itemize} \item Initialize \( \mathcal{C'} = \mathcal{C} \). \item While \( |\mathcal{C'}| > 1 \), compute the intermediate points iteratively: \begin{equation} \mathcal{C'} = \{ \text{lerp}(c_i, c_{i+1}, t) \mid i = 1, \dots, n-1 \} \end{equation} Equation (3) describes my implementation of the BezierCurbe::evaluateStep function, where the linear interpolation function is defined as: \begin{equation} \text{lerp}(c_1, c_2, t) = (1 - t)c_1 + t c_2. \end{equation} \item The final interpolated curve point \( \mathcal{C'} \) is added to \( \mathcal{B} \). \end{itemize}
\textbf{Return the computed Bézier curve points:} $ \mathcal{B} = \bigcup_{t \in (0,1)} \{\mathcal{C'}_t\} $ Of course, we cannot display at infinite resolution so, in practice, we need to sample at discrete values of t\in[0,1].
